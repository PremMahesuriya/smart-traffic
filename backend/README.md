# Smart Traffic Backend Service

This is a REST API backend service built using Node.js, Express, and MongoDB (via Mongoose) for the Smart Traffic Management System.

---

## 1. Folder Structure

```
backend/
├── server.js               # Express application and server runner
├── package.json            # NPM dependencies and runner scripts
├── .env                    # Dotenv secret keys & environment config
├── config/
│     database.js           # Database connection config
├── models/
│     TrafficLog.js         # Mongoose schema for traffic density counts
│     SignalLog.js          # Mongoose schema for adaptive signal states
├── controllers/
│     trafficController.js  # Traffic API route business logic
│     signalController.js   # Signal API route business logic
├── routes/
│     trafficRoutes.js      # Endpoint paths for /api/traffic
│     signalRoutes.js       # Endpoint paths for /api/signals
└── services/
      csvImporter.js        # CSV parsing and seeding service
```

---

## 2. Database Schemas

### TrafficLogs (`TrafficLog`)
- `timestamp` (Number): Video timeline position (seconds).
- `lane` (String): Lane indicator (`A`, `B`, `C`, `D`).
- `vehicleCount` (Number): Number of vehicles in frame.
- `density` (String): Classified density level (`Low`, `Medium`, `High`, `Very High`).
- `processingTime` (Number): Processing delay (milliseconds).

### SignalLogs (`SignalLog`)
- `timestamp` (Number): Video timeline position (seconds).
- `activeLane` (String): Lane with active signal (`A`, `B`, `C`, `D` or `None`).
- `signalState` (String): Active signal color state (`GREEN`, `YELLOW`, `RED`).
- `greenTime` (Number): Target duration assigned (seconds).
- `remainingTime` (Number): Timer countdown remaining (seconds).

---

## 3. REST API Endpoints

### Traffic API Paths
- `GET /api/traffic/current`: Returns counts and densities for all lanes in the latest frame.
- `GET /api/traffic/history`: Returns historical lane counts.
- `GET /api/traffic/analytics`: Returns aggregate summaries (Total Vehicles, Average Vehicles Per Lane, Busiest Lane, and Least Busy Lane).

### Signal API Paths
- `GET /api/signals/current`: Returns active lane, color state, and remaining countdown.
- `GET /api/signals/history`: Returns transition logs of all previous signal changes.

---

## 4. Seeding & Importing CSV Logs

To import the CSV log sheets generated by previous AI segments (Day, Night, Webcam, Heavy traffic) into MongoDB, run:
```bash
npm run import
```
This clears existing collections and parses all `*_lane_counts.csv` and `*_signal_logs.csv` logs inside the `output/` directory, importing them cleanly.

---

## 5. How to Run

1. **Prerequisites**: Ensure MongoDB is running locally at `mongodb://localhost:27017/`.
2. **Installation**: Install dependencies (already installed in workspace, else `npm install`).
3. **Execution**:
   ```bash
   # Run CSV log importer
   npm run import

   # Run server in development watch mode
   npm run dev

   # Run server in production mode
   npm run start
   ```

## 6. Example HTTP Requests (via curl / HTTP clients)

### Get Current Traffic
```bash
curl http://localhost:5000/api/traffic/current
```
**Response:**
```json
{
  "laneA": { "count": 1, "density": "Low" },
  "laneB": { "count": 2, "density": "Low" },
  "laneC": { "count": 2, "density": "Low" },
  "laneD": { "count": 1, "density": "Low" }
}
```

### Get Traffic Analytics
```bash
curl http://localhost:5000/api/traffic/analytics
```
**Response:**
```json
{
  "totalVehicles": 8,
  "averageVehiclesPerLane": 1.25,
  "busiestLane": "Lane C",
  "leastBusyLane": "Lane A"
}
```

### Get Current Signal
```bash
curl http://localhost:5000/api/signals/current
```
**Response:**
```json
{
  "activeLane": "A",
  "signal": "GREEN",
  "remainingTime": 11
}
```
